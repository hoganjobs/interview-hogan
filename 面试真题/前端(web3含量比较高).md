## 1. 一笔交易怎么确定需要多少 Gas？

### 标准答案：

- 使用 `eth_estimateGas` 或框架提供的 `contract.estimateGas.method(args)` 进行估算。
- 节点会在本地模拟执行交易，返回一个预估值。
- 实际执行时仍可能因为链上状态不同而略有差异。

**代码示例：**

```javascript
const gasEstimate = await contract.estimateGas.transfer(to, amount);
```

## 2. 模拟调用和正常调用有什么区别？

### 标准答案：

- **模拟调用（eth_call / callStatic）**

  - 只在本地执行，不会修改链上状态。
  - 不消耗 Gas，不需要签名。
  - 常用于测试交易是否能成功或获取只读数据。

- **正常调用（sendTransaction / writeContract）**
  - 需要用户签名和支付 Gas。
  - 会修改链上状态，上链后不可逆。

## 3. 发送交易之后，如何确定交易的状态？

### 标准答案：

- **通过交易哈希 txHash 查询：**

  - 调用 `provider.getTransactionReceipt(txHash)`。
  - 或使用 ethers.js `tx.wait()` 等方法。

- **返回值中包含：** 是否成功、区块号、消耗的 Gas、事件日志等。
- **前端可通过轮询或事件监听更新 UI 状态。**

---

## 4. 交易哈希是怎么得到的？

### 标准答案：

- 在执行写操作时（如 `contract.transfer()`），钱包会返回一个交易对象，其中包含 `hash` 字段。
- 交易哈希是对交易内容进行 RLP 编码并经过 keccak256 哈希后的结果，链上唯一标识该笔交易。

## 5. 如何批量调用多个合约读取请求？

### 标准答案：

- 使用 wagmi 的 `readContracts` 方法，一次性传入多个合约配置。
- 或者调用 Multicall 合约，将多笔只读请求打包执行。
- 返回结果可逐一解构并在前端展示。

---

## 6. 如何在项目中新增一条链？

### 标准答案：

- 在链配置中增加 `chainId`、RPC URL、`currency` 等参数。
- 在前端调用 wagmi / ethers 时指定新的链。
- 配置钱包连接逻辑，使其能切换或添加该链。

---

## 7. DEX 路由是如何处理的？

### 标准答案：

- 根据用户输入的 `amountIn` 或 `amountOut`，调用路由合约或聚合器。
- 聚合器返回最佳路径（包含多跳 swap）。
- 前端将最优路由展示给用户，并提供 Gas 成本、价格影响等信息。

## 8. ERC20 与 ERC721 的区别？

### 标准答案：

- **ERC20：** 同质化代币，可互换，每个单位没有差别。
- **ERC721：** 非同质化代币，每个 `tokenId` 唯一，常用于 NFT。
- **主要区别：** 元数据存储方式不同、所有权判定方式不同。

## 9. 如何判断代币是否被转走？

### 标准答案：

- **事件监听：**

  - ERC20 → `Transfer(address from, address to, uint256 value)`
  - ERC721 → `Transfer(address from, address to, uint256 tokenId)`

- **链上查询：**

  - ERC20 → `balanceOf(address)`
  - ERC721 → `ownerOf(tokenId)`

- **常见做法是事件监听 + 定期查询双保险。**

## 10. 监听交易事件的原理是什么？

### 标准答案：

- **区块链节点在新区块生成时，会将交易日志（logs）存储到链上。**
- **SDK 提供订阅功能：**

  - 订阅区块 → 扫描其中的交易。
  - 订阅合约事件 → 节点推送事件日志。

- **实现方式：**
  - WebSocket 订阅（实时推送）。
  - JSON-RPC `eth_getLogs` 轮询。

## 11. Solana 与 Ethereum 签名有什么区别？

### 标准答案：

- **Ethereum**

  - 签名算法：ECDSA (secp256k1)。
  - 地址：由公钥哈希得到，42 位 16 进制。
  - 签名数据：交易的 RLP 编码。

- **Solana**
  - 签名算法：Ed25519。
  - 地址：直接使用公钥（Base58）。
  - 签名数据：序列化后的交易消息。

## 12. 如何区分钱包适配器在多链中的封装？

### 标准答案：

- **底层：** 针对不同链分别封装（EVM → ethers/wagmi，Solana → Solana Web3.js）。
- **中间层：** 提供统一的 `connect`、`disconnect`、`signMessage` 接口。
- **顶层：** 对外暴露统一的 API，屏蔽链差异。
